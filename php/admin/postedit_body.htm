PostEdit

PostEdit is a php application that WaSQL uses to create pages and templates into a PostEdit folder on your local hard drive. This allows you to use any editor you wish to update your pages and templates. When the PostEdit Manager detects a file changed it checks for syntax and commits your changes to your WaSQL database.

If you have WaSQL on your local computer then the postedit program is already installed. If not, you will need to download it via git.

PostEdit requires a configuration file called postedit.xml. This file contains authentication information for each domain/website you want to connect to. Add the following entry to postedit.xml found in the postedit directory to authenticate to this domain as the current user:

<host name="<?=$_SERVER['HTTP_HOST'];?>" alias="<?=strlen($_SERVER['SUBDOMAIN'])?$_SERVER['SUBDOMAIN']:$_SERVER['HTTP_HOST'];?>" apikey="<?=userValue('apikey');?>" username="<?=userValue('username');?>" groupby="name" />

Possible host attributes and their explanations are as follows (red attributes are required):

  • name - this is the hostname you want to connect to. It should correlate to the host name in your config.xml file
  • username - the username to authenticate as. This must be a valid username for this domain.
  • apikey - the apikey for the authenticating user. This is found in the user profile menu after logging in. Changing your username or password will change your apikey.
  • alias - This gives a more friendly alias to the hostname. For instance, dev.wasql.com may have an alias of wasql.
  • insecure - If your website is not HTTPS then set this attribute to 1.
  • tables - tables to download locally so you can modify them. This defaults to "_pages,_templates".
